# Модуль "SNiP Validator"

Этот модуль представляет собой изолированную систему для проверки проектов перепланировки на соответствие строительным нормам и правилам (СНиП) с использованием ИИ-агентов.

## 1. Назначение

Модуль использует 10 параллельно работающих ИИ-агентов на базе `Langchain` и `Ollama` (модель `gemma3:27b`) для анализа текстового описания проекта (`doc.md`) на основе набора правил, изложенных в `SNiP.md`.

## 2. Установка

Перед использованием убедитесь, что у вас установлен Python 3.9+ и запущен сервер Ollama с загруженной моделью `gemma3:27b`.

1.  **Установите зависимости:**
    Перейдите в директорию `snip` и выполните команду:
    ```bash
    pip install -r requirements.txt
    ```

## 3. Использование

### 3.1. Запуск проверки

Для запуска анализа документа `doc.md` выполните главный скрипт:

```bash
python main.py
```

Скрипт запустит 10 агентов, каждый из которых проанализирует документ и выведет свой вердикт в консоль.

### 3.2. Проверка обновлений

Для симуляции ежедневной проверки обновлений базы знаний `SNiP.md` можно запустить:

```bash
python update_checker.py
```

**Настройка автоматического запуска:**

Для автоматической ежедневной проверки можно настроить `cron` (в macOS/Linux). Откройте редактор `crontab -e` и добавьте строку (путь нужно будет изменить на абсолютный):

```cron
0 3 * * * /usr/bin/python /path/to/your/project/homo/snip/update_checker.py >> /path/to/your/project/homo/snip/update.log 2>&1
```
Эта команда будет запускать скрипт каждый день в 3:00 ночи.
